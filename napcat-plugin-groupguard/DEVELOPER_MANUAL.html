GroupGuard Developer Manual (v1.0.23)

Technical documentation for the NapCat GroupGuard plugin.

1. Architecture Overview

The plugin follows a monolithic architecture designed for the NapCat (OneBot 11) framework. It operates as a single Node.js module that hooks into the NapCat event lifecycle.

3. Key Modules

3.1 Entry Point (src/index.ts)

  • plugin_init: Initializes PluginState, loads config, connects DB, starts `AuthManager`, and registers Express routes for WebUI.
  • plugin_onmessage: The main message loop.
    1. Checks License (`authManager.getGroupLicense`).
    2. Checks White/Blacklists.
    3. Dispatches to handleCommand.
    4. Runs passive checks (Anti-Recall, Filter, Spam, etc.).
  • plugin_onevent: Handles non-message events.
    • request.group.add -> GroupJoinVerifier
    • notice.group_increase -> Welcome & Verify
    • notice.group_recall -> Anti-Recall

3.2 Command Handler (src/commands.ts)

Contains all business logic for commands. It parses raw_message and executes actions via `pluginState.callApi`.

Note: Most commands require Admin or Owner permissions. Permission checks are performed inside each command block using isAdminOrOwner().

3.3 Authorization (src/auth.ts)

Manages the commercial licensing system.

  • Model: licenses: Map<GroupId, { level: 'pro'|'enterprise', expireTime: number }>
  • Granting: Adds duration to current expiration (cumulative).
  • Checking: checkFeature(groupId, featureName) validates if a group has access to specific Pro features.

3.4 Data Persistence (src/db.ts)

A lightweight JSON-based database with write-debouncing.

  • Structure: data/groups/{groupId}/{type}.json (e.g., activity.json, signin.json).
  • Caching: Data is cached in memory (`cache` object) and flushed to disk every 5 seconds or on critical updates.

5. Known Issues & Limitations

  • Curfew Logic Missing: The enableCurfew config exists, but the enforcement logic (checking time on message) appears to be missing in plugin_onmessage in src/index.ts.
  • Sync I/O: Config saving uses fs.writeFileSync. While safe for small scales, high-concurrency writes might cause slight event loop blocking.
  • Memory: msgCache (Anti-Recall) cleans up every 5 minutes, but extremely active groups might spike memory usage temporarily.

6. Development Tips

  • Debug Mode: Send 开启调试 to the bot (Private Chat, Owner only) to enable verbose logging in pluginState.logBuffer.
  • WebUI Dev: The WebUI is served statically. You can edit webui/config.html directly and refresh the browser (no rebuild needed for UI changes).
  • Adding Commands: Add new logic in src/commands.ts inside handleCommand. Ensure you return true if the command is handled to stop further processing.
