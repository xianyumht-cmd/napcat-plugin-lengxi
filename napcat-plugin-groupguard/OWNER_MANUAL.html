部署与运维

1. 基础环境

本插件基于 NapCat (OneBot11) 运行。推荐使用 Docker 部署。

docker run -d -e ACCOUNT=123456789 -p 3000:3000 -p 3001:3001 --name napcat mlikiowa/napcat-docker:latest

2. 安装插件

将编译好的 napcat-plugin-groupguard.zip 解压到 NapCat 的 plugins 目录。

3. 初始配置

首次运行后，请修改 config/napcat-plugin-groupguard/config.json 或使用 WebUI。

  • 设置 ownerQQs：您的QQ号（最高权限）。
  • 设置 licenseKey：商业授权密钥（可选）。

商业授权管理

所有授权指令仅限 主人QQ 通过 私聊 机器人执行。

指令 说明 示例
授权 <群号> <天数> 为指定群开启授权（专业版） 授权 123456 30
授权 <群号> 永久 为指定群开启永久授权（企业版） 授权 123456 永久
回收授权 <群号> 立即停止指定群的服务 回收授权 123456
查询授权 <群号> 查看群授权状态及剩余时间 查询授权 123456

全局风控系统 (Anti-Ban)

风控设置是 全局生效 的，所有群都会应用此策略。
仅限主人私聊设置。

随机后缀 (Random Suffix)

在所有发出的消息末尾追加不可见字符，防止内容重复被检测。

开启随机后缀
关闭随机后缀

随机延迟 (Random Delay)

发送消息前随机等待一段时间，模拟真人操作。

设置随机延迟 <最小ms> <最大ms>

例: 设置随机延迟 500 1500

自身撤回 (Auto Recall Self)

机器人发出的消息在指定时间后自动撤回，减少历史记录留存。

开启全局自身撤回
关闭全局自身撤回

全局黑白名单

拉黑 <QQ> 全局拉黑，该用户在所有群无法触发指令
取消拉黑 <QQ> 解除全局拉黑
白名单 <QQ> 全局白名单，豁免所有检测（刷屏、违禁词等）
取消白名单 <QQ> 解除全局白名单
添加全局违禁词 <词> 设置所有群通用的违禁词
添加全局拒绝词 <词> 设置所有群通用的入群拒绝关键词

高级功能 (仅限主人)

多群广播 <内容> 向所有已授权的群发送通知消息（自动延时防风控）
开启/关闭调试 开启或关闭详细日志输出（Debug Mode）
