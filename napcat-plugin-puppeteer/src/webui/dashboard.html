API 路径说明

推荐 无认证 API（供其他插件调用）
{host}/plugin/napcat-plugin-puppeteer/api/{endpoint}

例如：http://localhost:6099/plugin/napcat-plugin-puppeteer/api/render

WebUI 需认证 API（WebUI 管理）
{host}/api/Plugin/ext/napcat-plugin-puppeteer/{endpoint}

需要 WebUI 登录 Token，用于配置管理等操作

其他插件调用示例

在 NapCat 插件中调用渲染服务（使用无认证 API）：

POST /render - HTML 模板渲染 
// 在其他插件中调用（无需认证）
const response = await fetch('http://localhost:6099/plugin/napcat-plugin-puppeteer/api/render', {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify({
        html: '<div style="padding:20px;background:#fff;"><h1>Hello {{name}}</h1></div>',
        data: { name: 'World' },
        encoding: 'base64'
    })
});
const result = await response.json();
// result.data 为 Base64 编码的图片数据
POST /screenshot - URL 截图 
const response = await fetch('http://localhost:6099/plugin/napcat-plugin-puppeteer/api/screenshot', {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify({
        file: 'https://example.com',
        file_type: 'url',
        encoding: 'base64',
        setViewport: { width: 1280, height: 800 }
    })
});
const result = await response.json();
// result.data 为 Base64 编码的截图

提示

请将 localhost:6099 替换为实际的 NapCat 服务地址和端口。插件间通信无需认证，可直接调用。

核心服务

POST
/screenshot 无认证

通用截图接口，支持 URL、本地文件路径或直接传入 HTML 字符串进行渲染。

Request Body (JSON)
参数名 类型 必填 说明
file string 是 目标内容 (URL / HTML代码 / 文件路径)
file_type string 否 指定内容类型: url | htmlString | file | auto(默认)
selector string 否 CSS 选择器，只截取指定元素。默认 body
omitBackground boolean 否 是否隐藏默认背景（设为透明）。默认 false
data object 否 Handlebars 模板数据，仅当 file 为模板时有效
waitSelector string 否 等待该元素出现后再截图
setViewport object 否 { width, height, deviceScaleFactor }
Response JSON
{
  "code": 0,
  "data": "Base64String...",  // 图片数据
  "message": "OK",
  "time": 150                 // 耗时(ms)
}
POST
/render 无认证

/screenshot 的语义化别名，专门用于 HTML 模板渲染。

参数名 类型 必填 说明
html string 是 HTML 模板字符串
data object 否 模板插值数据
GET /screenshot

轻量级 URL 截图接口，适合快速调试或简单场景。

Query Parameters
参数名 必填 说明
url 是 目标网页地址
width 否 视口宽度 (默认 1280)
height 否 视口高度 (默认 800)
selector 否 元素选择器
raw 否 如果为 true，直接返回 image/png 流，不包装 JSON

浏览器控制

GET
/browser/status 无认证

获取 Puppeteer 实例的详细状态。

{
  "code": 0,
  "data": {
    "connected": true,      // 浏览器是否连接
    "version": "Chrome...", // 版本信息
    "pageCount": 1,         // 打开的页面数
    "pid": 12345,           // 进程 ID
    "executablePath": "..." // 浏览器路径
  }
}
POST
/browser/{action} 需认证

生命周期控制接口，支持 start, stop, restart。

🔒 此接口需要 WebUI 认证 Token
POST /browser/start POST /browser/stop POST /browser/restart 
{ "code": 0, "message": "Browser started successfully" }

系统配置

POST
/config 需认证

热更新插件配置，实时生效（部分浏览器配置需重启生效）。

🔒 此接口需要 WebUI 认证 Token
Request Body
{
  "browser": {
    "headless": true,
    "args": ["--no-sandbox"]
  },
  "maxPages": 10,
  "lockTimeout": 30000
}
GET
/status 无认证

获取插件整体运行统计。

{
  "totalRenders": 100,
  "failedRenders": 2,
  "uptimeFormatted": "2小时 15分"
}

注意事项

  • 修改浏览器配置后需要重启浏览器才能生效
  • 最大页面数设置过高可能导致内存占用过大
  • 自定义浏览器路径需确保路径正确且有执行权限
